Chapter 1. Getting started Docker

Remarks

Docker is an open-source project that automates the deployment of application inside software containers. These application containers are similar to lightweight virtual machines, as they can be run in isolation to each other and the running host.
Docker requires features present in recent linux kernels to function properly, therefore on Mac OSX and Windows host a virtual machine running linux in required for docker to operate properly.
Currently the main method of installing and setting up this virtual machine is via Docker ToolBox that is using VirtualBox internally, but there are plans to integrate this functionality into docker itself, using the native virtualisation features of the operating system. On Linux systems docker run natively on the host itself.

Installing Docker on Mac OSX

Requirements: OS X 10.8 "Mountain Lion" or newer required ti run Docker.
While the docker binary can run natively on Mac OS X, to build and host containers you need to run Linux virtual machine on the box.
1.12.0
Since version 1.12 you don't need to have a separate VM to be installed, as Docker can use the native Hypervisor.framework funtionality of OS X to start up a small Linux machine to act as backend. To install docker follow the following steps.

  • Go to Docker for Mac.
  • Download and run the installer.
  • Continue through installer with default options and enter your account credentials when requested.
More information
To verify installation:
  • Open the Docker.app from the Application folder, and make sure it is running. Next open up Terminal.

Installing docker on Windows

Requirements: 64 bits version of Windows 7 or higher on a machine which supports Hardware Virtualization Technology and it is enabled.
While the docker binary can run natively on Windows, to build and host containers you need to run a Linux virtual machine on the box.
1.12.0
Since version 1.12 you don't need to have a separate VM to be installed, as Docker can use the native Hyper-V functionality of Windows to start up a small Linux machine to act as backend.
To install docker follow the following steps.

  • Go to Docker for Windows.
  • Download and run the installer.
  • Continue through installer with default options and enter your account credentials when requested.
Check here for more information on the installation.
To verify installation:
  • Start Docker from the Start menu if it hasn't been started yet, and make sure it is running Next upen up any terminal (either cmd or PowerShell)

Installing docker on Ubuntu Linux

Docker is supported on the following 64 bits of Ubuntu Precise 12.04(LTS) Linux or higher.
A couple or notes:

The following instruction involve installation using Docker packages only, and this ensure obtaining the lastest official release of Docker. If you need to install only using Ubuntu-managed packages, consult the Ubuntu documentation (not recommended otherwise for obvious reasons). Ubuntu Utopic 14.10 and higher exist in Docker's APT repository but exist official supported and not no longer official supported due to known security issues.
Prerequisites
  • Docker only works on a 64 bits installation of Linux.
  • Docker requires Linux Kernel version 3.10 or higher (except for Ubuntu Precise 12.04, which requires version 3.13 or higher). Ckeck current kernel version with the command:
    uname -r
    Check this post if yo need to update your Ubuntu kernel by scrolling further down. Ubuntu installations.
Update APT sources
This needs to be done so as to access packages from Docker repository.
  • Log into your machine as a user with sudo or root privileges.
  • Open a terminal window.
  • Update package information, ensure that APT works with the https method, and that CA certificates are installed.
$ sudo su
$ apt-get update
$ apt-get install \
ca-certificates \
curl \
software-properties-common
  • Add Docker's Official GPG key.
$ curl -fsSL https://download.docker.com/linux/ubuntu/gpg | sudo apt-key add -
